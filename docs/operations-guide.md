# Marine LMS: эксплуатация

## Локальный запуск

```powershell
cd "C:\Users\телеграм 8\Documents\LMS"
npm.cmd install
npm.cmd run build
npm.cmd run start
```

По умолчанию приложение слушает `http://127.0.0.1:3000`.

Для другого порта:

```powershell
$env:PORT="3001"
npm.cmd run start
```

## Основные переменные окружения

- `HOST` - адрес bind, по умолчанию `127.0.0.1`.
- `PORT` - порт, по умолчанию `3000`.
- `PUBLIC_BASE_URL` - публичный адрес для QR-кодов и ссылок в письмах.
- `MAX_MATERIAL_UPLOAD_MB` - лимит обычного учебного файла, по умолчанию `25`.
- `MAX_VIDEO_UPLOAD_MB` - лимит видеофайла, по умолчанию `512`.
- `MAX_PHOTO_UPLOAD_MB` - лимит фото студента, по умолчанию `3`.

## SMTP

Минимальная настройка:

```powershell
$env:SMTP_HOST="smtp.example.com"
$env:SMTP_PORT="587"
$env:SMTP_STARTTLS="true"
$env:SMTP_FROM="noreply@example.com"
$env:SMTP_USER="noreply@example.com"
$env:SMTP_PASS="password"
npm.cmd run start
```

Проверка в интерфейсе: `Админ -> Уведомления -> Проверить SMTP`.

Если SMTP не настроен, уведомления не теряются: они сохраняются в журнале со статусом `Лог`.

## Резервные копии

Перед импортом, массовой привязкой видео, обновлением или переносом:

```powershell
npm.cmd run backup:data
```

В backup должны попадать:

- `data/db.json`
- `data/uploads`
- `imports/wordpress/output`

Минимальное правило: хранить хотя бы ежедневные копии за 7 дней и отдельную копию перед каждым крупным импортом.

## Безопасность

Что уже включено:

- CSRF-токены для авторизованных POST-форм.
- Same-origin проверка POST-запросов.
- Rate limit на попытки входа.
- Смена пароля в профиле студента.
- Восстановление пароля через временный пароль и уведомления.
- Раздельные лимиты загрузки фото, учебных файлов и видео.
- Аудит последних 2000 административных POST-действий.

Где смотреть аудит: `Админ -> Аудит`.

## Роли админ-панели

- `admin` - полный доступ ко всем разделам и действиям.
- `instructor` - ограниченный доступ: может открыть `Админ -> Пользователи`, создать студента, редактировать его данные, загрузить фото и назначить курс.

Инструктор не может:

- удалять или архивировать пользователей;
- удалять курсы, уроки, материалы, назначения или попытки;
- вручную выдавать сертификаты;
- смотреть отчеты, тесты, сертификаты, файлы, уведомления и аудит;
- редактировать курсы и настройки.

Создать инструктора может полный администратор через `Админ -> Пользователи -> Создать пользователя -> Роль`.

## Админские сценарии

Студент:

1. `Админ -> Пользователи`.
2. Открыть `Профиль студента`.
3. Проверить обязательные поля и фото.
4. Назначить курс или вручную выдать сертификат.

Обучение:

1. `Админ -> Отчеты`.
2. Отфильтровать по курсу или статусу.
3. При необходимости сбросить попытки или разблокировать пересдачу.

Главная страница:

1. `Админ -> Главная`.
2. Отметить курсы, которые должны быть видны на первой странице.
3. Указать порядок отображения.
4. Нажать `Сохранить витрину`.
5. На публичной главной нажать `Подробнее`, чтобы проверить страницу описания курса.
6. Открыть `/courses`, чтобы проверить полный каталог всех активных курсов.

Сертификаты:

1. `Админ -> Сертификаты`.
2. Найти сертификат по номеру, студенту или курсу.
3. Открыть, скачать PDF, проверить QR, отозвать или перевыпустить.
4. Скачать `Экспорт CSV` или `Excel-реестр` по текущим фильтрам.
5. Проверить `Журнал действий по сертификатам`: выдача, отзыв, перевыпуск и повторная отправка.

Шаблон сертификата:

1. `Админ -> Курсы -> Редактировать`.
2. В блоке `Шаблон сертификата` изменить HTML или загрузить `.html` файл.
3. Нажать `Открыть предпросмотр`, чтобы увидеть образец без создания сертификата.

Файлы и видео:

1. `Админ -> Файлы`.
2. Проверить битые ссылки и непривязанные видео.
3. Привязать видео вручную или запустить автопривязку по названию.

## Перенос на сервер

Минимальный порядок:

1. Скопировать проект.
2. Выполнить `npm.cmd install`.
3. Скопировать `data/db.json` и `data/uploads`.
4. Настроить `PUBLIC_BASE_URL` и SMTP.
5. Запустить `npm.cmd run build`.
6. Запустить `npm.cmd run start` через process manager сервера.

Перед публичным запуском желательно перейти с JSON-файла на полноценную БД и настроить автоматические backup.
